## 01 コメント誘導型Reel（最重要・最高実績）

### 成功事例2本の比較

| | 投稿A | 投稿B |
|---|---|---|
| URL | https://www.instagram.com/reel/DTIN5hIiTLa/ | https://www.instagram.com/reel/DT5Pn8sieCp/ |
| 誘導ワード | **シュート** | **核心** |
| いいね | **8,900** | 5,500 |
| コメント | **1,400** | 272 |
| 訴求軸 | 親しみ・低摩擦 | 「真実を教える」知的訴求 |
| 保護者反応 | 少 | あり（「高校生の娘に見せたかった」） |

### 2本から導いた成功法則

**法則1：誘導ワードの摩擦が低いほどコメント数は増える**
- 「シュート」はバスケ経験者なら反射的に打てる単語
- 「核心」は意味を理解してから打つ→1ステップ増える→5倍以上の差
- コメント数を最大化したいときは **日常語・競技用語** を使う

**法則2：「核心系ワード」はリード質が高い**
- 「核心」コメントには保護者・本気勢が多い
- いいね数は少なくてもLINE転換率が高い可能性がある
- LINEリスト拡大フェーズ → 「シュート」型 / リード質重視フェーズ → 「核心」型 で使い分ける

**法則3：コメント欄が荒れても止めない**
- 「シュート入らない人はバスケ向いてない」等の否定コメントも拡散に貢献
- 反論コメントがつくほどアルゴリズムには有利に動く
- 反応が来たら削除せず、丁寧に返答する（ブランド毀損にならない範囲で）

### 誘導ワード選定フレームワーク

```
目的がLINE登録数最大化 → 「シュート」「スリー」「フォーム」（最低摩擦・競技用語）
目的がリード質向上    → 「核心」「本質」「秘密」（知的好奇心訴求）
目的がブランド認知    → 「ASCENT」「診断」（サービス名直結）
```

### 再現テンプレート

```
【動画構成（コメント数最大化版：シュート型）】
0〜2秒：インパクトのある映像 or テキストフック（止まらせる）
2〜8秒：シュートの技術 or 課題感を刺激するシーン
8〜12秒：画面テキスト「シュートが入らない原因、知りたい？」
最後1秒：「コメントに『シュート』と書いて」

【動画構成（リード質重視版：核心型）】
0〜2秒：「シュートが入らない本当の理由、誰も教えてくれない」
2〜15秒：一般論を否定し、Kaitだけが知る本質を一部公開
15〜25秒：「全部知りたい人は『核心』とコメントしてみてください」

【キャプション共通構成】
1行目：フック（問い or 断言）
2〜3行目：空白
4〜8行目：技術的価値提供（3〜4行）
最終行：コメント誘導CTA
```

### 運用ルール（コメント誘導型）
- 誘導ワードはサービスに直結する単語（シュート・スリー・フォーム・核心 等）
- 「プレゼント」「無料」などの訴求は不要。コンテンツ自体が価値になるように設計
- コメントDM自動送信はLステップで設定（キーワードトリガー）
- 投稿頻度：週1〜2本を安定運用（量より再現性）
- A/Bテスト：同じ動画内容で誘導ワードだけ変えて効果測定する

### ⚠️ 重要制約：バイラル投稿単体では購買転換しない

**実績データ（2026年4月時点）**
- 投稿A（シュート）：いいね8,900・コメント1,400 → 購買転換 ≒ 0
- 投稿B（核心）：いいね5,500・コメント272 → 購買転換 ≒ 0

**原因：バイラル投稿が引き寄せる層は「購買意欲のない観客」**
- 「シュート」「核心」に反応するのはバスケに興味ある人全般
- ASCENTの購買者は「本気で改善したい中高生 or その保護者」という絞られた層
- コメント誘導型はリーチ最大化に特化しており、リード質のフィルタリングができない

**結論：バイラル投稿の正しい役割はLINE登録数の最大化のみ**
- Instagram → LINE登録 → ここで終わらせてはいけない
- LINE登録後の「ナーチャリング設計」が購買を決める

## 02 購買ファネル設計（最重要）

### 現状の断絶ポイント

```
Instagram（バイラル） → LINE登録 → [ここで断絶] → 購買
```

バイラル投稿はLINE登録まで機能している可能性が高い。
購買ゼロの原因は「LINE登録後の設計」にある。

### 購買が起きるファネルの構造

```
【STEP 1】Instagram（認知・LINE登録）
  └ バイラルReel → コメント → DM → LINE登録
     役割：量的なリスト拡大のみ。質は問わない

【STEP 2】LINE（教育・信頼構築）← ここが現状の空白
  └ 登録直後：問題意識を特定するメッセージ
  └ 3〜7日：「なぜ改善できないか」の構造解説
  └ 7〜14日：ASCENTの診断で何が変わるかの具体提示
  └ 14日〜：Trial誘導CTA
     役割：「観客」を「本気の改善希望者」に絞り込む

【STEP 3】Trial（体験・意思確認）
  └ trial.stepbystep-ascent.com での無料体験
     役割：価値を体感させ、課金への心理障壁を除去する

【STEP 4】購買（LITE/UNLEASH/PRO）
  └ 体験後のフォローメッセージ → プラン選択
```

### LINE登録直後シナリオ（要設計・現在未整備）

登録直後のメッセージが購買率を最も左右する。以下を設計・実装すること。

```
【登録直後メッセージ】
「シュートを本気で変えたいと思っているなら、
まず1つだけ聞かせてください。

今、シュートで一番困っていることは何ですか？
① フォームが安定しない
② 試合になると入らない
③ 練習では入るのに本番でダメ
④ そもそも基礎から見直したい

番号を送ってみてください」
```

→ 回答に応じて課題別のナーチャリングシナリオに分岐させる

### 購買に近いリードの特徴（ターゲット再定義）

バイラル投稿経由ではなく、以下の行動をとる人が購買に近い：

- プロフィールのLINEリンクを自発的にクリックした人
- 「フォーム見てもらえますか」「診断について教えてください」とDMしてきた人
- Trialページに到達した人
- 保護者で「子どもに使わせたい」と問い合わせた人

**この層へのアプローチを最優先にする。バイラル数字を追わない。**

## 03 Reel構成パターン（ASCENTブランド準拠）

### パターンA：課題提示→原因→解決（教育型）
```
0〜3秒：「シュートが入らない理由はこれだ」（テキストフック）
3〜12秒：よくある間違いを映像で見せる
12〜20秒：正しいフォームや意識を見せる
20〜25秒：「詳しく知りたい人はプロフのリンクから」
```

### パターンB：Before/After（変化型）
```
0〜2秒：「練習1ヶ月後の変化」（テキスト）
2〜15秒：選手のBefore映像
15〜25秒：After映像と変化のポイント
25〜30秒：コメント誘導またはLINE誘導
```

### パターンC：問いかけ型（共感型）
```
0〜3秒：「こういうシュートになってない？」（問いかけ）
3〜15秒：あるある映像（ターゲットが共感する失敗シーン）
15〜25秒：「なぜそうなるか」の解説
25〜30秒：CTA（コメント or LINE）
```

## 03 キャプション設計ルール

### 構造
```
1行目：フック（数字・問い・断言のどれか）
2〜3行目：空白（展開させるため）
4〜8行目：価値提供（技術情報 or 共感 or 事例）
最終行：CTA（1つだけ）
```

### フックの型（使い回し可）
- 「〇〇が入らない選手の9割がやってること」
- 「シュート確率を上げる、たった1つの感覚」
- 「俺が気づくまで3年かかったこと」（創業者ボイス）
- 「中高生に今すぐ伝えたいこと」

### CTAの型
- 「気になる人は『シュート』とコメントしてみてください」（コメント誘導）
- 「詳しくはプロフのLINEから無料で受け取れます」（LINE誘導）
- 「まず無料で試してみたい方はリンクから」（Trial誘導）

## 04 LINE配信スキル

### 配信タイミング最適解
- 月曜 20:00〜21:00（週明け練習モチベが高い時間帯）
- 木曜 19:00〜20:00（週末の練習前）

### 配信構成テンプレート
```
【件名】シュートで悩んでいる人へ

こんにちは、Kaitです。

（1〜2行：共感フック）
→「最近こういう相談が増えています」

（3〜4行：価値提供）
→「実はシュートが入らない原因の8割は〇〇です」

（1行：CTA）
→「詳しくはこちらで確認してみてください」
[ボタン：今すぐ確認する]
```

### 禁止事項（claude.mdコピーライティングルール準拠）
- 命令形NG（「やってください」→「やってみてください」）
- 24時間以内・限定・絶対 等の根拠なし訴求NG
- 1配信に中高生と保護者を同時ターゲットにしない

## 05 LP改善スキル

### 最重要指標
- Trial申し込みCVR（目標：訪問者の3〜5%）
- LINEクリック率（目標：配信数の15%以上）

### CVR改善チェックリスト
- [ ] ファーストビューに「誰のためのサービスか」が3秒で伝わるか
- [ ] CTAボタンが動詞始まりになっているか（「今すぐ無料で〜する」）
- [ ] 料金前に価値提示ができているか
- [ ] スマホ表示でCTAが画面内に収まっているか
- [ ] FAQが丁寧語統一されているか

## 06 施策判断基準

このスキルで提案する施策は必ず以下を満たすこと：

1. **再現性があるか** → 1回限りでないか
2. **スケール可能か** → Kait不在でも動くか
3. **数字で検証できるか** → いいね数・CVR・LINE登録数で測定可能か
4. **Kaitにしかできないか** → AIで代替できるなら自動化を優先する
