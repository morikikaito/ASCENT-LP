## 診断フロー（全フェーズ）

```
PHASE 0：初回チェック
  └─ localStorage に ascent_trial_used フラグがあるか確認
     ├─ あり → 「診断済み」画面 + UNLEASH誘導 表示
     └─ なし → PHASE 1へ

PHASE 1：動画アップロード
  └─ ユーザーが動画をアップロード
     ↓
  Gemini API → 動画バリデーション（バスケシュートか否か）
     ├─ NO → 「バスケのシュート動画を送ってください」メッセージ
     └─ YES → PHASE 2へ

PHASE 2：基本スペック入力
  └─ フォーム表示（年齢・身長・体重・性別・利き手・送信者）
     ↓ 入力完了 → PHASE 3へ

PHASE 3：診断処理中表示
  └─ 「AIが1フレームずつ解析中...1〜2分お待ちください」
     ↓
  Gemini API（動画 + スペック + システムプロンプト）→ 診断開始
     ↓ → 5択質問（1問目）を返す

PHASE 4：5択質問 × 3回
  └─ 1問ずつ表示 → 回答 → 次の質問
     ↓ 3回完了 → PHASE 5へ

PHASE 5：最終診断結果表示
  └─ 原因○個の提示
     ├─ 1つ目の原因のみ詳細表示（体験版は非VIP扱い）
     └─ 残りは「会員限定」として隠す
     ↓
  localStorageに ascent_trial_used: true を保存
     ↓
  UNLEASH誘導（固定バナー）
```

# 4. 診断用マスターデータ：ASCENT最新理論

【最重要アップデート1：新・シュートミート理論（骨盤正対ステップ）】
・「骨盤をリングに正対させること」を最優先のゴールとする。従来の「足を並行（パラレル）にする」という固定概念は完全に廃止。
・移動しながらミートする場合、慣性によって骨盤が流されるため、「2歩目の足」の位置でこれを強制的に補正する。
・リングに正対している場合、そのままスクエアで構える。移動時は2歩目の足を「半歩前」に出し骨盤のズレをロックする。
・場所によって足の位置を変える：トップ・0度はパラレル（平行）、45度の場合そのサイド側の足を1歩前に出す。

【最重要アップデート2：クワトロコア理論（体幹の4分割操作）】
・体幹部を4つのコア（右上重心＝右肩甲骨、左上重心＝左肩甲骨、右下重心＝右骨盤、左下重心＝左骨盤）に分割し、それぞれを独立して稼働させる。
・膝指導・腕指導の手打ちを避け、コアを起点として連動させる。
・チェックポイント：腰で姿勢を作っていないか（胸椎9番あたりを使えているか）。膝主導になっていないか。骨盤（仙腸関節）から動けているか。肩甲骨を起点に腕を動かせているか。胸椎9番あたりを反らせ、お腹の奥の筋肉（大腰筋）がピンと張る感覚を作る。

【ASCENT 3指標（世界唯一）】
01 リリース連動角度：肩→肘→手首の直線性をAIが算出。骨格3点から直接計算。
02 連動タイムラグ（★世界唯一）：下半身→上半身への力の伝達時間をフレーム単位で計測。
03 骨盤正対スコア（★世界唯一）：シュートミート時の腰の向きを数値化。腰骨の傾き角度で計算。
